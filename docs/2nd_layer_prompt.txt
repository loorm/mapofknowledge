**TASK: Extend knowledge_map.html to render a second layer of emergent knowledge from knowledge_map_emergent.json as a visually distinct layer above the base layer, connected by vertical edges — as shown in the reference image at `design/layered_design_example.jpg`.**

**Please look at the reference image first before reading further. It shows the visual effect and concept we want, minus the colours, shapes the glows.**

**CONTEXT:**
The current HTML renders a 2D force-directed network of ~2,500 nodes (the base layer). We want to add a second layer of emergent/interdisciplinary knowledge fields that floats visually above the base layer, connected to it by vertical lines dropping down to the base layer nodes they draw from. The overall effect is two distinct horizontal planes with connecting vertical lines between them — like a layered architecture diagram.

**The key insight from the reference image:**
- Layer 1 (base) is a flat 2D network spread across its own horizontal plane
- Layer 2 (emergent) is another flat 2D network on a plane above it
- The "draws from" connections are vertical lines descending from emergent nodes down to base layer nodes
- Within each layer, nodes connect horizontally to their own children/parents as normal

**IMPLEMENTATION APPROACH:**

Since D3 force simulation is inherently 2D, simulate the two layers separately rather than trying to do true 3D. The illusion of depth is achieved through:

1. **Vertical offset**: Render all emergent layer nodes with a fixed Y offset applied — shift them upward on screen by a substantial fixed amount (e.g. 600-800px) relative to the base layer centre. This creates the visual separation between planes.

2. **Two separate force simulations**: Run one simulation for the base layer and one for the emergent layer. Each settles independently. Do not mix nodes from both layers in the same simulation.

3. **Draws_from edges as vertical connectors**: After both simulations have settled, draw lines from each emergent node down to the base layer node it draws from. These lines should be:
   - Nearly vertical (they will have a slight angle because nodes won't be perfectly aligned)
   - Rendered in a glowing cyan or light blue, semi-transparent
   - Visually prominent enough to read as "this field emerges from that foundation"
   - Not part of either simulation — just drawn in screen space based on node positions

4. **Perspective hint (optional but nice)**: Apply a very subtle CSS or SVG transform to give a faint isometric/perspective tilt to the overall canvas — just enough to suggest 3D depth without distorting readability. If this proves complex, skip it and rely purely on vertical separation.

Emergent layer (`knowledge_map_emergent.json`): string node IDs prefixed "E", `layer: 2`, edges with `edge_type: "hierarchical"` or `edge_type: "draws_from"`. Draws_from edges point to base layer integer IDs.
